# Quote Form Application

This project is a React application built with Vite, designed to generate quotation forms. It leverages WebAssembly (WASM) for PDF generation, Tailwind CSS for styling, and ESLint for code quality.

## Features

-   **React + Vite**: Modern and fast development setup for a React application.
-   **WebAssembly (WASM)**: Integrates Go-compiled WebAssembly for efficient PDF generation directly in the browser.
-   **Tailwind CSS**: Utility-first CSS framework for rapid UI development.
-   **ESLint**: Configured for maintaining code quality and consistency.
-   **Custom Hooks**: Provides `useWasm` for WASM initialization and `useSubmit` for form submission and PDF handling.

## Packages Used

The core dependencies and development tools include:

-   **`react`**: JavaScript library for building user interfaces.
-   **`vite`**: Next-generation frontend tooling.
-   **`@vitejs/plugin-react`**: Vite plugin for React projects.
-   **`@tailwindcss/vite`**: Vite plugin for Tailwind CSS integration.
-   **`vite-plugin-wasm`**: Vite plugin to handle WebAssembly modules.
-   **`eslint`**: Pluggable JavaScript linter.
-   **`eslint-plugin-react-hooks`**: ESLint rules for React Hooks.
-   **`eslint-plugin-react-refresh`**: ESLint plugin for React Fast Refresh.

## Getting Started

### Prerequisites

Make sure you have Node.js (and npm or yarn) installed on your system.

### Installation

1.  Clone the repository:
    ```bash
    git clone <repository-url>
    cd quoteform
    ```
2.  Install dependencies:
    ```bash
    npm install
    # or
    yarn install
    ```

### Development Server

To run the development server:

```bash
npm run dev
# or
yarn dev
```

The application will be available at `http://0.0.0.0:5173`. The `vite.config.js` is configured to listen on `0.0.0.0` and port `5173`, which is useful for accessing the development server from other devices on the same network (e.g., via Tailscale).

### Build for Production

To build the application for production:

```bash
npm run build
# or
yarn build
```

This will generate a `dist` directory containing the production-ready static assets.

### Deployment

The `dist` folder generated by the build command can be served by any static file server.

## Core Logic

### WebAssembly Integration (`src/hooks/Wasm.js`)

The `initWasm` hook handles the loading and initialization of the `main.wasm` module. It expects a global `window.Go` object (typically loaded via a script tag in `index.html` for Go-compiled WASM). Once loaded, it instantiates the WASM module and runs it, making Go functions available in the global scope (e.g., `window.generatePDF`).

### Form Submission and PDF Generation (`src/hooks/Submit.js`)

The `useSubmit` hook manages the form submission process. It checks if WASM is ready, formats the input data, and then calls the `window.generatePDF` function (provided by the WASM module) with the payload. It handles different WASM response types (Uint8Array or Base64 string) to create a PDF blob and opens it in a new tab.

### Default Values (`src/Default.js`)

This file defines default structures and initial values for the quotation form, including helper functions like `getToday` and `getValidUntil`.
